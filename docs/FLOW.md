## GymLog (가칭) — 화면 흐름도 v1

### 목적
- MVP 기능이 실제 사용자 흐름으로 어떻게 이어지는지 한 장으로 설명한다.
- 와이어프레임 발표/공유 시 "어디서 무엇을 하는지"를 빠르게 전달한다.

---

## 1) 메인 사용자 흐름 (기록 생성 중심)

`세션 목록 화면`
- 과거 운동 기록을 **요약**으로 볼 수 있음(운동일, 제목, 종목 수·세트 수 등)
-> `새 세션 만들기` 클릭  
-> `세션 생성 화면` (운동일/제목/메모 입력, 제목 자동값: YYYY-MM-DD 운동)  
-> `세션 상세 화면`  
- **종목·세트 전체 내용**은 세션 상세에서 확인한다.
-> `종목 추가` 클릭 → **화면 전환**으로 `운동 종목 목록/등록 화면` 진입 (부위별 조회, 필요 시 종목 등록)  
-> `세션 상세 화면` 복귀  
-> 종목별 `세트 추가` (원판 중심 입력)  
-> 저장/자동 반영  
-> `세션 목록 화면` 복귀

---

## 2) 조회 흐름 (기록 확인 중심)

`세션 목록 화면`  
-> 세션 카드 선택  
-> `세션 상세 화면`  
-> 종목 블록 및 세트 묶음 표시 확인  
(예: 같은 무게/횟수 연속 시 `n세트`로 표시)

---

## 3) 수정 흐름

`세션 상세 화면`  
-> 제목/메모 수정  
-> 종목 블록 추가/삭제  
-> 세트 추가/수정/삭제  
-> 변경 즉시 반영

정책:
- 세션 내 종목/세트 삭제: 확인 대화상자 생략(MVP)
- 세트 라벨(워밍업/피더/탑/백오프): MVP 필수 항목 아님

---

## 4) 삭제 흐름

### 4-1. 세션 전체 삭제
`세션 목록 화면` 또는 `세션 상세 화면`  
-> 세션 삭제  
-> 확인 메시지 1회 권장  
-> 삭제 완료

### 4-2. 라이브러리 종목 삭제
`운동 종목 목록/등록 화면`  
-> 종목 삭제  
-> 라이브러리 목록에서는 제거  
-> 과거 세션 화면에는 기록 당시 종목명 유지(스냅샷)

---

## 5) 화면-기능 ID 매핑

- 세션 목록 화면: `F-001`, `F-007`
- 세션 상세 화면: `F-002`, `F-004`, `F-005`, `F-006`
- 세션 생성 화면: `F-003`
- 운동 종목 목록/등록 화면: `F-008`, `F-009`, `F-010`

---

## 6) 30분 내 발표용 포인트

1. 사용자는 "목록 -> 생성 -> 상세 -> 종목 선택 -> 세트 입력" 순서로 기록을 완성한다.
2. 종목은 라이브러리로 재사용하고, 세트는 한 줄 단위로 저장한다.
3. 표시와 저장을 분리한다. (UI 묶음 표시 가능, 저장은 세트별)
4. 삭제 정책은 MVP에서 단순하되, 과거 기록 가독성은 유지한다.
