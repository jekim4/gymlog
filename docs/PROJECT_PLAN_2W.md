# GymLog — 2주 프로젝트 계획 (통합 와이어프레임 기준)

**기준 문서:** `gymlog_integrated_wireframe.pdf`  
**구현 저장소:** `gym-log/` (Next.js + Prisma + SQLite)  
**추출 텍스트:** `gymlog_wireframe_extract.txt`

> **기본 트랙 = Lite (약 22시간)** — 평일 1시간 + 주말 3시간 × 2주.  
> 하루 2~3시간을 넣을 수 있으면 맨 아래 **Standard 트랙**을 함께 보면 된다.

| 구간 | 계산 | 합계 |
|------|------|------|

| 주말 4일 × 3시간 | 12h | |

**운영 원칙**

- **주말(3h):** 새 페이지·DB·Server Action처럼 **한 덩어리**만 끝낸다.
- **평일(1h):** 전날/주말 작업 **이어하기**, 스타일·문구·작은 버그만.
- 한 번에 Day 6~10 전체를 끝내려 하지 않는다.

## 1. Lite — 2주 후 목표 (Must)

PDF 플로우 중 **핵심 루프**만 2주에 넣는다.

| 완료 기준 | 설명 |
|-----------|------|
| 메인 허브 | 월 이동, 운동한 날 도트, 선택일 요약·종목 리스트(간단 표시) |
| 기록 작성 | 선택일 세션 생성, `[+ 운동]` → 종목 선택 → 세션 상세 |
| 세션 상세 | 제목·메모, 종목 블록, 세트 추가/수정/삭제, 세션 삭제(확인) |
| 종목 선택 | 부위 탭 7종 + 리스트 + `[+]` 후 상세 복귀 |
| 마무리 | `[✓ 오늘의 운동 완료]` → 메인 복귀, 프리웨이트 시드 + 샘플 1일 |

**2주에서 빼고 3주차로 미룸 (Lite Won’t)**

| 항목 | 이유 |
|------|------|
| **루틴★ / Routine / ▶ 주입** | 구현·시드·스키마 부담 큼 → **Week 3** |
| **총부하 자동 계산** (4-7) | 상세에 원판·횟수만 먼저 → **Week 3** |
| Start 저항 UI | 있으면 좋음, 없어도 Lite 완료 가능 |
| 연속 세트 묶음 표시 (4-8) | 백로그 |
| 타이머·검색·피드 | 백로그 |

## 2. Lite — 2주 캘린더 일정

총 **22시간**을 블록으로 나눈다. `(평)` `(주)` = 그날 권장 투자 시간.

### 1주차 — 메인 허브 + DB

| 날 | 블록 | 시간 | 할 일 | 끝났을 때 |
|----|------|------|--------|-----------|
| **토 (주)** | W1-A | 3h | 라우트 맵(`/`, `/sessions/[id]`, `/exercises/select`), `/records` → 메인 통합 방향 결정, 메인 UI 골격(캘린더·월 네비) | 화면만 떠도 됨 |
| **일 (주)** | W1-B | 3h | `Session` 날짜별 조회·생성, 메인에 선택일 요약·종목 리스트 **DB 연동** | 날짜 바꾸면 DB 반영 |
| 월 (평) | W1-C | 1h | 메인 도트·빈 상태 문구, `/dev` 대시보드 분리 등 정리 | |
| 화 (평) | W1-C | 1h | `[+ 운동 종목 추가]` → `/exercises/select` 이동만 | 링크 동작 |
| 수 (평) | W1-C | 1h | 상세 페이지 빈 껍데기 `/sessions/[id]` | |
| 목 (평) | W1-C | 1h | 오늘 세션 없을 때 자동 생성 로직 | |
| 금 (평) | W1-C | 1h | 버그 수정, 스크린샷 1장 | **W1 마일스톤** |

**1주차 마일스톤:** 메인에서 **날짜 선택 → 그날 기록(요약) 보기 → 오늘 세션 존재**.

---

### 2주차 — 상세·세트·종목 선택

| 날 | 블록 | 시간 | 할 일 | 끝났을 때 |
|----|------|------|--------|-----------|
| **토 (주)** | W2-A | 3h | 세션 상세: 제목·메모 편집, 종목 블록 목록, 세션 삭제(confirm) | 상세 조회·편집 |
| **일 (주)** | W2-B | 3h | 세트 테이블 CRUD(순번·원판 kg·횟수), `+ 세트 추가` | 세트 저장됨 |
| 월 (평) | W2-C | 1h | `/exercises/select` 부위 탭 7종 + 리스트 | |
| 화 (평) | W2-C | 1h | `[+]` → `SessionExercise` 생성 → 상세 복귀 | 종목 추가 루프 |
| 수 (평) | W2-C | 1h | 프리웨이트 시드 + 샘플 세션 1일 | 데모 데이터 |
| 목 (평) | W2-C | 1h | `[✓ 오늘의 운동 완료]` → 메인 복귀 | |
| 금 (평) | W2-C | 1h | 전체 플로우 점검·README 한 줄 | **Lite 2주 완료** |

**2주차 마일스톤:**  
`메인 → [+ 운동] → 종목 [+] → 상세에서 세트 입력 → [완료] → 메인`

## 4. Lite — 데모 시나리오 (2주 금요일)

1. 메인: 이번 달 이동, 운동한 날 **도트** 확인  
2. **오늘** 선택 → 요약·종목 카드  
3. `[+ 운동]` → **등** 탭 → 종목 `[+]` → 상세 이동  
4. 세트 2줄 입력·저장  
5. `[✓ 오늘의 운동 완료]` → 메인 복귀  
6. (선택) Prisma Studio로 행 확인  

※ **루틴★ / ▶** 는 3주차 데모에 추가.
