# GymLog (가칭) — 와이어프레임 v1 (확정본 초안)

## 문서 역할

- MVP 화면 **구조·흐름·정책**을 한 곳에 모은 **v1 확정용** 문서다.
- 구현 시 참고 순서: 본 문서 → `FUNCTIONAL_REQUIREMENTS.md` → `FUNCTIONAL_REQUIREMENTS_TABLE.md` → `FLOW.md`
- `WIREFRAME_TEMPLATE.md`는 작성 과정용 초안이며, **v1 기준은 본 문서**로 본다.

## 전체 화면 맵 (MVP)

1. **세션 목록** — 과거 세션 탐색, 새 세션 진입, 세션 삭제(확인)
2. **세션 상세** — 그날 종목·세트 조회·편집, 종목/세트 추가
3. **세션 생성** — 운동일·제목(자동)·메모 후 저장 → 상세로 이동
4. **운동 종목 라이브러리** — 부위별 목록, 종목 등록·삭제(과거 기록 이름 유지)

---

## 화면 1. 세션 목록

### 목적

최근 세션을 빠르게 훑고, **한 세션의 상세**로 들어가거나 **새 세션**을 만든다.

### 연결 기능 ID

`F-001`, `F-007`

### 주요 구성

| 구역 | 내용 |
|------|------|
| 헤더 | 제목 `운동 세션 목록`, 정렬 `최신순`(MVP 고정 표시 가능) |
| 리스트 | 카드 단위: 운동일, 제목, 종목 수, 세트 수(요약) |
| 액션 | 카드 탭 → 상세, `삭제`(세션 전체, 확인 1회 권장), `+ 새 세션 만들기` |
| 빈 상태 | 안내 문구 + 첫 세션 만들기 |

### 목록 vs 상세 (표시 범위)

- **목록:** 운동일, 제목, 종목 수·세트 수 등 **요약**만으로 충분하다.
- **상세:** 그 세션의 **종목 블록·세트 줄**까지 전부 본다. (`FLOW.md`와 요구사항을 이 기준으로 맞춘다.)

### 정책·예외

- 세션 **전체 삭제** 전에는 확인 메시지 1회 권장.
- 세션 **없음**이면 빈 상태 UI.

## 화면 2. 세션 상세

### 목적

선택한 날의 기록을 **종목 단위 블록**으로 보고, 제목·메모·종목·세트를 수정한다.

### 연결 기능 ID

`F-002`, `F-004`, `F-005`, `F-006`

### 주요 구성

| 구역 | 내용 |
|------|------|
| 헤더 | 뒤로(목록), 운동일, (선택) 더보기·세션 삭제 |
| 세션 카드 | 제목(수정 가능), 메모(선택) |
| 종목 블록 | 종목명, **기본 저항(Start)** 표시(종목에 등록된 경우), 세트 리스트 |
| 세트 한 줄 | 저장 단위: **무게(원판 중심) + 횟수 + 순번** (문자열이 아닌 필드로 구현 시 동일 의미) |
| 액션 | `+ 종목 추가`, `+ 세트 추가`, 종목 블록 삭제, 세트 줄 삭제·수정 |

### 표시 규칙 (UI)

- **저장:** 세트는 1세트 1행 원칙.
- **표시:** 연속된 세트가 **무게·횟수가 동일**하면 `n세트` 묶음 표기 가능.
- **무게·횟수 중 하나라도 다르면** 다른 줄로 표시.
- 워밍업/피더/탑/백오프 **라벨은 MVP 필수 아님.**

### 머신 Start vs 세트 입력

- **Start:** 운동 종목(라이브러리 또는 스냅샷)에 둔다. 라벨 문구는 `Start 3.6kg` 또는 `기본 저항 3.6kg` 등으로 통일 가능.
- **세트 입력:** 기본은 **원판만** 기록. 필요 시 UI에서 `Start + 원판` 합산 표시.

### 정책·예외

- 세션 **안에서** 종목 블록·개별 세트 삭제 시 **확인 생략**(MVP).
- 세션 **전체 삭제**는 확인 1회 권장(목록 또는 상세에서 제공 시 동일).

### 레이아웃 스케치 (텍스트)

```
[뒤로]  세션 상세          [더보기]
┌─────────────────────────────┐
│ 2026-04-08  푸시 데이  [수정] │
│ 메모: 가슴+어깨 …             │
└─────────────────────────────┘
┌ 종목: 해머 인클라인 프레스      │
│ Start 3.6kg                  │
│ • 원판 0kg × 15 × 1          │
│ • 원판 2.5kg(양) × 15 × 2    │
│ [+세트] [종목 삭제]            │
└─────────────────────────────┘
[+ 종목 추가]
```

## 화면 3. 세션 생성

### 목적

새 세션의 **운동일·제목·메모**만 받고 저장한 뒤 **세션 상세**로 보낸다.

### 연결 기능 ID

`F-003`

### 주요 구성

| 입력 | 규칙 |
|------|------|
| 운동일 | 필수, 기본값 오늘 |
| 제목 | 자동 `YYYY-MM-DD 운동`, 사용자 수정 가능 (예: 푸시 데이) |
| 메모 | 선택 |

| 버튼 | 동작 |
|------|------|
| 저장 | 검증 후 세션 생성 → **세션 상세**로 이동 |
| 취소 | 목록으로 복귀(입력 버림) |

### 정책·예외

- 운동일 미입력 시 저장 불가(안내).
- 저장 직후 종목·세트는 비어 있어도 된다(상세에서 추가).

---

## 화면 4. 운동 종목 라이브러리 (목록 / 등록)

### 목적

자주 쓰는 머신·프리웨이트를 **재사용 카드**로 등록·조회·삭제한다.

### 연결 기능 ID

`F-008`, `F-009`, `F-010`

### 주요 구성

| 구역 | 내용 |
|------|------|
| 헤더 | `운동 종목 라이브러리`, `+ 등록` |
| 부위 탭 | 가슴 / 등 / 어깨 / 팔 / 하체 / 코어 (MVP 고정) |
| 리스트 | 종목명, 브랜드(선택), 기본 저항 Start(선택), 행 액션: 선택·삭제 |
| 등록 폼 | 이름(필수), 부위(필수), 브랜드(선택), Start(선택), 저장 |

### 정책·예외

- **같은 동작이라도 브랜드·기종이 다르면 별도 종목**으로 둔다.
- **라이브러리에서 종목 삭제**해도, **과거 세션에는 기록 당시 종목명**이 남는다(스냅샷 등 구현 방식은 개발 단계에서 확정).
- 시스템 **기본 프리웨이트** 종목의 사용자 삭제 허용 여부는 별도 정책(`F-010`).

### 세션 상세와의 연결 (MVP 확정)

- `세션 상세`에서 `+ 종목 추가` 시: **전체 화면 전환**으로 본 화면(`운동 종목 라이브러리`)에 진입한다.  
- 종목 선택(또는 등록) 후 **다시 세션 상세로 복귀**한다. 모달·바텀시트는 MVP 범위에서 쓰지 않는다.

## 화면 간 이동 (요약)

| 출발 | 행동 | 도착 |
|------|------|------|
| 세션 목록 | 새 세션 | 세션 생성 → 저장 후 세션 상세 |
| 세션 목록 | 카드 선택 | 세션 상세 |
| 세션 상세 | 종목 추가 | 종목 라이브러리(전체 화면 전환) → 선택 후 세션 상세 복귀 |
| 세션 상세 | 세트 추가 | 동일 화면 내 폼/행 추가 |
| (탭/메뉴) | 종목 관리 | 종목 라이브러리 |

상세 흐름은 `FLOW.md` 참고.
