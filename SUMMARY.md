## GymLog (가칭) — 진행 요약

### 목표
- 운동 기록 웹 서비스를 만들며 **TypeScript 기반 풀스택 개발 역량**을 키운다.
- 요구사항 분석 → 설계 → 개발 → 테스트 → 배포 → 유지보수의 **전체 사이클**을 단계적으로 학습한다.

---

### 작업용 앱 이름
- `GymLog (가칭)`
- 후보: GymLog / SetNote / LiftJournal / Reps

---

### MVP 범위(1인 사용 기준)
- 사용자는 날짜 기준으로 **운동 세션**을 만들고, 세션 내에 **운동 종목과 세트 기록(무게·횟수·세트 순번)** 을 남긴다.
- 사용자는 세션을 **목록(최신순)** 으로 보고, 선택한 세션의 **상세 내역**을 조회할 수 있다.
- 사용자는 세션/세트 정보를 **수정**하고 필요 시 **삭제**할 수 있다.
- 사용자는 **운동 종목 라이브러리**를 관리(등록/조회)하고, 기본 종목 데이터(프리웨이트 일부)를 제공받는다.

---

### 핵심 결정사항(설계 원칙)
- **종목(라이브러리)** 과 **기록(세션/세트)** 을 분리한다.
  - 종목은 재사용 가능한 “카드”
  - 세션은 날짜 단위 기록
  - 세트는 숫자(무게/횟수) 단위 기록
- 같은 동작명이라도 **브랜드/기종이 다르면 별도의 종목**으로 관리한다.
  - 예: 바디마스터 숄더프레스, 해머스트렝스 숄더프레스
- 운동 종목 선택 UX는 **부위별(등/가슴/어깨/하체 등) 분류**를 MVP의 기본 구조로 채택한다.
- 삭제 정책은 MVP에서 단순화한다.
  - 세션 삭제 시 해당 세션의 종목/세트가 함께 삭제
  - MVP에서는 **물리 삭제**(소프트 삭제는 확장 아이디어로 보류)
